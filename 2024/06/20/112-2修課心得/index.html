112-2修課心得

發表於 2024-06-20 更新於 2024-06-21 分類於 修課心得

資管系必修

資訊管理導論

  • 授課教授：彭志宏教授
  • 上課時間：禮拜一下午
  • 上課方式：
    • 教授用簡報上課
    • 助教教Tableau、Text mining、ER diagram
    • 請講師來分享
    • 小組輪流上台報告
  • 評分方式：
    • Assignments (e.g., Group Assignmentsand Tutorials) 40%
    • Group Project 45%
    • Class Participation 15%
      但說實在的他根本沒有公布細項評分，我只拿到最後的等第而已
  • 上課用書：完全不會用到課本
  • 甜度：★★★★☆
    我Class Participation 只有1分然後有一次 In class exercise 只有50分最後拿A
  • 涼度：★★★☆☆
    有很多雜七雜八的事要做，期末要做出個像樣的東西出來要花滿多時間的
  • 推薦程度：★☆☆☆☆
    不是必修就不用特別來修了
  • 課程內容：
    學期初教授會叫你填自我介紹的表單，我到學期末還是不知道這個表單的用意。然後教授會介紹一些資訊管理相關的知識，我只記得提款機會先吐卡再吐錢，其他都不記得了。助教上課會教一些工具，然後會有In class exercise，通常有在聽或者旁邊的同學有在聽就沒問題。今年還有一個Social Media Campaign要我們推廣茶文化，我完全不知道教授希望我們學會什麼，但總而言之我們拍了兩隻challenge的短片。學期初教授就會叫我們趕快開始準備期末專案的事，但我覺得其實也不用真的那麼趕，除非你想要做出很厲害的東西，不然拖到期中後再來處理也沒什麼問題，但主題要慎選，選一些好發揮的，我覺得我們選美髮業是不太明智的選擇
  • 作業及考試：
    • In class exercise * 7
    • Group exercise * 4
    • Social Media Compaign presentation * 2
    • Final Project proposal and Final Project presentation

作業系統

  • 授課教授：陳建錦教授
  • 上課時間：禮拜二早上
  • 上課方式：
    • 教授用簡報上課
  • 評分方式：
    • 期中考 35%
    • 期末考 35%
    • 程式作業 30%
  • 上課用書：作業系統經典恐龍書，但我連PDF都沒有載完全只看簡報
  • 甜度：★★★★☆
    照原始分數算，無情不調分，我84.3拿A-
  • 涼度：★★★☆☆
    我常常翹課，也不點名，但補影片補得滿累的，建議還是可以去上個課
  • 推薦程度：★★★★☆
    教授教得不錯，但我沒學好
  • 課程內容：
    教授大致上用簡報上課，有時候有些補充的講義會放COOL，上課都會錄影然後又不點名，原則上沒有去上課的必要，但教授教得不錯然後又很帥可以去看一下
  • 作業及考試：
    • Programming Assignment * 3
    • Midterm exam
    • Final exam

統計學一下(會計系)

  • 授課教授：陳郁蕙教授
  • 上課時間：禮拜二下午
  • 上課方式：
    • 教授用他挖空很多講義上課
  • 評分方式：
    • 期中考 35%
    • 期末報告 35%
    • 平時成績 30%
  • 上課用書：沒有
  • 甜度：★★★★☆
    我期中考55最後A-，因為也沒有給細項的成績所以我不確定到底甜不甜，但應該是滿甜的
  • 涼度：★★☆☆☆
    不太涼因為講義挖空很多，沒抄看不太懂，然後會偷點名
  • 推薦程度：★★☆☆☆
    教授教得不錯，但是期中考要我把那些公式跟檢定方法背起來真的不可能欸教授
  • 課程內容：
    教授用他的講義上課，會寫滿整個白板。教授會稱呼學生「親愛的」，上課的方式很像媽媽，時不時會提起往事。每週都有實習課，就是去聽助教講怎麼用python作檢定統計還有迴歸分析，我都沒在聽，反正有交就有分
  • 作業及考試：
    • Python 作業 * 2
    • 期中作業 (手寫)
    • 期末作業 (手寫)
    • 期中考
    • 期末報告
